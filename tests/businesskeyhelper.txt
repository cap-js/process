
// Example that shows a prefix is required to differentiate processes

new Car: {
    ID: "1234",
    mileage: 100
}
Process carProcess Start with context: {
    businesskey: "1234"
    ID: "1234"
    mileage: 100
}


new Dog: {
    ID: "1234",
    name: "Alex"
}
Process dogProcess Start with context: {
    businesskey: "1234"
    ID: "1234"
    name: "Alex"
}


update Dog with name = "Tim"

--> Processes with businesskey "1234" will be cancelled --> both processes

Possible Solution: 
- prefix businessKey with Entity Name
- introduce new annotation to let the developer customize the prefix
